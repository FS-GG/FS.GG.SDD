Artifact traceability walkthrough for 001-sdd-artifact-model
============================================================

Source of truth
---------------

The source of truth for the first artifact-model slice is the public contract
defined by:

- specs/001-sdd-artifact-model/data-model.md
- specs/001-sdd-artifact-model/contracts/artifact-schemas.md
- specs/001-sdd-artifact-model/contracts/work-model-json.md
- specs/001-sdd-artifact-model/contracts/lifecycle-rule-contracts.md
- specs/001-sdd-artifact-model/contracts/fixture-catalog.md

The implementation exposes the same trace through
LifecycleArtifacts.standardArtifactContracts().

SDD-owned artifacts
-------------------

.fsgg/project.yml
  Purpose: project identity and lifecycle roots.
  Structured contract: schemaVersion 1, project id, default work root, SDD
  config path, agent config path, optional Governance pointers.
  Generated view relationship: contributes project summary and source digest to
  readiness/<id>/work-model.json.
  Stale behavior: source digest mismatch is reported through staleGeneratedView.
  Diagnostic family: missingArtifact, malformedSchemaVersion.

.fsgg/sdd.yml
  Purpose: lifecycle stages, artifact layout, generated-view policy, migration
  posture.
  Structured contract: schemaVersion 1 plus lifecycle, artifacts,
  generatedViews, and migrations sections.
  Generated view relationship: contributes lifecycle policy facts to work model
  and later analysis/verify/ship views.
  Stale behavior: source digest, schema version, or generator drift is reported
  through staleGeneratedView.
  Diagnostic family: missingArtifact, malformedSchemaVersion,
  unsupportedSchemaVersion.

.fsgg/agents.yml
  Purpose: Claude/Codex guidance targets generated from lifecycle data.
  Structured contract: schemaVersion 1, agent targets, source model, projection
  policy.
  Generated view relationship: feeds readiness/<id>/agent-commands/.
  Stale behavior: generated guidance is not authoritative; stale projections
  are diagnostics.
  Diagnostic family: missingArtifact, staleGeneratedView.

work/<id>/spec.md
  Purpose: work metadata, requirements, decisions, and user-facing scope.
  Structured contract: schemaVersion 1 front matter plus stable FR/DEC ids.
  Generated view relationship: sources workItem, requirements, decisions, and
  source digest entries in work-model.json.
  Stale behavior: generated views must match the current spec digest.
  Diagnostic family: missingArtifact, requirementNotTyped,
  proseStructuredMismatch.

work/<id>/tasks.yml
  Purpose: typed implementation task graph.
  Structured contract: schemaVersion 1 task ids, owner, dependencies,
  requirement/decision refs, skills, and evidence refs.
  Generated view relationship: sources task graph entries in work-model.json.
  Stale behavior: generated views must match the current tasks digest.
  Diagnostic family: missingArtifact, duplicateIdentifier, unknownReference,
  workModelInconsistent.

work/<id>/evidence.yml
  Purpose: implementation and verification evidence declarations.
  Structured contract: schemaVersion 1 evidence ids, kind, subject, artifacts,
  result, and synthetic marker.
  Generated view relationship: sources evidence entries in work-model.json.
  Stale behavior: generated views must match the current evidence digest.
  Diagnostic family: missingArtifact, unknownReference, workModelInconsistent.

readiness/<id>/work-model.json
  Purpose: deterministic normalized lifecycle contract.
  Structured contract: schemaVersion, modelVersion, workId, project, sources,
  workItem, requirements, decisions, tasks, evidence, generatedViews,
  diagnostics, governanceBoundaries.
  Generated view relationship: generated from SDD sources and consumed by tools,
  tests, agents, and optional Governance.
  Stale behavior: staleGeneratedView when source digests or generator metadata
  no longer match.
  Diagnostic family: staleGeneratedView, malformedDigest.

Optional Governance boundaries
------------------------------

.fsgg/policy.yml, .fsgg/capabilities.yml, and .fsgg/tooling.yml are exposed as
owner = governance, requiredBySdd = false, relationship =
optionalCompatibilityBoundary. SDD records the references but does not evaluate
route selection, freshness, profile adjustment, gate enforcement, audit, or
release behavior.

Fixture trace
-------------

The fixture corpus under tests/fixtures/sdd-artifact-model/ covers:

- valid-work-item: zero blocking diagnostics.
- malformed-schema-version: malformedSchemaVersion and unsupportedSchemaVersion.
- missing-artifact: missingArtifact with required paths.
- duplicate-identifiers: duplicateIdentifier for requirement, decision, task,
  and evidence ids.
- unknown-reference: unknownReference for graph references.
- prose-structured-mismatch: proseStructuredMismatch while structured data
  remains executable truth.
- stale-generated-view: staleGeneratedView for stale generator metadata.
- deterministic-ordering: byte-identical JSON across repeated normalization.
