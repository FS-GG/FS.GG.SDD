# Contract: Gate-Artifact Regeneration Semantics

**Status**: authoritative for `fsgg-sdd checklist` and `fsgg-sdd tasks` re-runs.
**Consumers**: SDD authors and agents; the authoring-contracts skill mirrors this (pinned by
a drift guard). No persisted-schema version change.

A **gate artifact** (`work/<id>/checklist.md`, `work/<id>/tasks.yml`) is generated by the
tool *and* carries a small amount of author-owned content. Its content splits into:

- **Tool-owned (derived) content** â€” recomputed from the current authoritative sources on
  every run. In `checklist.md`: the *Source Snapshot, Checklist Items, Review Results,
  Accepted Deferrals, Blocking Findings* sections. In `tasks.yml`: every task row's
  structure (`id`, `SourceIds`, `Title`, dependencies, requirement/decision refs, required
  skills/evidence) and file-level derived findings.
- **Author-owned content** â€” preserved across regeneration. In `checklist.md`: the *Source
  Specification, Source Clarifications, Advisory Notes, Lifecycle Notes* sections. In
  `tasks.yml`: each task's `status` and `owner`, plus file-level advisory/lifecycle notes.

## Rules

1. **Never re-ingest tool-owned content as input.** On a re-run, tool-owned content is
   re-derived from the current sources â€” never carried forward from the prior file's rows.
   A derived row exists in the output **iff** the current sources justify it.
2. **Always re-derive, regardless of the source snapshot.** The recorded source snapshot is
   provenance, not a gate on re-derivation. A re-run recomputes tool-owned content whether or
   not the snapshot matches.
3. **Preserve author-owned content.** Authored sections are left byte-untouched. Authored
   per-task `status`/`owner` are carried forward onto the re-derived task that shares the
   same identity (leading `SourceId`; the stable `T###` id is preserved on a match).
4. **Reclaim hand-edited derived rows.** If an author edits a tool-owned row by hand, the
   re-derived value overwrites it. Tool-owned rows are not an authoring surface.
5. **Idempotence.** With unchanged sources and no authored edits, a re-run produces a
   byte-identical artifact and a `noChange` outcome. No accumulation or duplication.
6. **The only re-run block is the `unsafe-overwrite` opt-out.** An artifact carrying
   `<!-- fsgg-sdd: unsafe-overwrite -->` is not overwritten; the existing `unsafeOverwrite`
   diagnostic names the exact file to delete and the command to re-run. No other path leaves
   an artifact silently stale, and no supported recovery requires inferring `rm`.

## Non-goals

- No per-row provenance markers or managed-region sentinels (unnecessary; see research
  Decision 1).
- No Governance runtime, verdict, evidence-freshness evaluation, or gate enforcement.
- No persisted-schema version bump.
