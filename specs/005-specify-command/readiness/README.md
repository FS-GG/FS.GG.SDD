# Readiness Evidence: Specify Command

This directory records real evidence for `fsgg-sdd specify`.

- `build.txt`: Release build.
- `specify-create-tests.txt`: focused specify create/rerun/dry-run tests.
- `specify-rerun-tests.txt`: focused safe rerun and refusal tests.
- `specify-diagnostics-tests.txt`: invalid request and diagnostic tests.
- `generated-view-tests.txt`: generated work-model currency tests.
- `traceable-output-tests.txt`: deterministic JSON, text projection, and Governance boundary tests.
- `command-workflow-tests.txt`: MVU read/effect boundary tests.
- `fsi-specify-public-surface.txt`: public FSI/prelude exercise.
- `specify-cli-smoke.txt`: disposable-directory CLI smoke output.
- `full-test-suite.txt`: complete test suite.
- `governance-boundary-review.txt`: SDD/Governance boundary review.
- `artifact-traceability.txt`: final task-to-artifact traceability.
- `performance.txt`: local command-test performance check.
- `human-summary-review.txt`: text summary review.

All evidence uses real filesystem or process paths where safe. No synthetic
dependencies are used for this feature.
