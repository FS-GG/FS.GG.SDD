# `governance-handoff.json` Schema Contract (schemaVersion 1, contract v1.0.0)

The Governance handoff is a **generated readiness view**, emitted to
`readiness/<id>/governance-handoff.json` by `fsgg-sdd ship` and regenerated by
`fsgg-sdd refresh`. It is the SDD-owned machine contract; see
[integration-requirements.md](integration-requirements.md) for the field→consumer
mapping and the cross-repo version pin. Markdown renderings (if any) are
projections over this JSON (Constitution II).

It carries **declared SDD facts only**: no effective/`autoSynthetic` evidence
state, no selected route/profile/gate, no severity or enforcement verdict
(FR-005, FR-009, SC-005).

## Determinism

Object keys and array elements are emitted in a fixed, sorted order through the
existing canonical `Serialization`. The document excludes implicit clocks,
durations, host absolute paths, directory-enumeration order, and ANSI styling
(FR-004, SC-003). Identical source trees produce byte-identical output.

## Document shape

```jsonc
{
  "schemaVersion": 1,
  "contractVersion": "1.0.0",
  "generatorVersion": "fsgg-sdd/<semver>",
  "workId": "017-governance-handoff",

  // Provenance: contributing sources + digests for stale detection (FR-003).
  "sources": [
    { "path": "readiness/017-governance-handoff/work-model.json",
      "digest": "sha256:…", "schemaVersion": 1 },
    { "path": "readiness/017-governance-handoff/verify.json",
      "digest": "sha256:…", "schemaVersion": 1 },
    { "path": "readiness/017-governance-handoff/ship.json",
      "digest": "sha256:…", "schemaVersion": 1 }
  ],

  // Evidence projection → Kernel.Evidence.build(nodes, dependencies). FR-005/006.
  "evidence": {
    "nodes": [
      { "id": "evidence:E001", "state": "real", "rationale": null },
      { "id": "evidence:E002", "state": "synthetic", "rationale": "stub data" },
      { "id": "task:T001", "state": "real" },
      { "id": "task:T002", "state": "pending" }
    ],
    "dependencies": [
      { "dependent": "evidence:E001", "dependency": "task:T001" },
      { "dependent": "task:T001", "dependency": "task:T002" },
      { "dependent": "task:T001", "dependency": "evidence:E002" }
    ]
  },

  // Routing enrichment → F015 Routing.route inputs (optional; Governance MAY
  // sense paths itself via F016). Paths are F014-normalized, repo-relative.
  "governedReferences": [
    { "path": "src/FS.GG.SDD.Artifacts/GovernanceHandoff.fs",
      "owner": "sdd", "relationship": "authored", "kind": "source",
      "operation": "modified" }
  ],

  // .fsgg config presence/pointers → F014 loaders. Absent = false, pointer omitted.
  "governanceConfig": {
    "policyPresent": false,
    "capabilitiesPresent": false,
    "toolingPresent": false
  },

  // Merge-boundary readiness facts → merge fence / future SDD-readiness gate.
  // Advisory inputs to a Governance decision, NOT a verdict (FR-008).
  "readiness": {
    "shipDisposition": "ship-ready",
    "verificationReadiness": "verified",
    "counts": { "advisory": 0, "warning": 0, "blocking": 0 },
    "blockingDiagnosticIds": [],

      { "view": "work-model.json", "state": "current" },
      { "view": "verify.json", "state": "current" },
      { "view": "ship.json", "state": "current" }
    ]
  },

  // Stale / missing / partial-config / cycle diagnostics (Constitution VIII).
  "diagnostics": []
}
```

## Field rules

- **`evidence.nodes[].state`** ∈ `{"pending","real","synthetic","failed","skipped"}`.
  `"autoSynthetic"` is **invalid** in a produced handoff (computed-only on the
  consumer side). Tokens are identical to `Kernel.Json` evidence tokens.
- **`evidence.dependencies[]`** edges are `(dependent, dependency)` = "dependent
  rests on dependency". Both endpoints MUST appear in `nodes` (else the consumer's
  `build` returns `UnknownNode`).
- **`rationale`** is present for `synthetic`/`skipped` nodes when SDD has one
  (`EvidenceEntry.Rationale`); `null`/omitted otherwise.
- **`readiness.*`** mirror `ShipSummary`/`VerificationSummary`; they never assert a
  pass/fail verdict.
- **`governanceConfig`** is always present; all-false with omitted pointers when no
  `.fsgg` Governance files exist (FR-011).
- **`diagnostics[]`** uses the SDD `Diagnostic` shape (`id`, `severity`, `message`,
  `correction`, `relatedIds`).

## Currency

The handoff is a `GenerationManifest` view of kind `GovernanceHandoff`. It is
reported `current`/`missing`/`stale`/`malformed` by comparing `sources[]` digests
against the live sources; `refresh` regenerates it. A present file is never assumed
current (FR-012).

## Forbidden content (boundary-exclusion, asserted by tests — SC-005)

A produced handoff MUST NOT contain: any `"autoSynthetic"` state; a selected
route, capability verdict, matched glob, or gate id; a profile name; a severity
verdict or effective-enforcement decision; or a ship/merge pass/fail verdict.
