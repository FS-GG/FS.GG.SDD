# Native SDD lifecycle commands readiness evidence

This directory records implementation evidence for the native command feature.

Current slice:

- `fsi-draft-public-surface.txt`: FSI transcript for the public command API.
- `init-interpreter.txt`: real interpreter evidence for `fsgg-sdd init`.
- `init-cli-smoke.txt`: disposable-directory CLI smoke output.
- `build.txt`: solution build output.
- `command-workflow-tests.txt`: focused MVU workflow test output.
- `init-command-tests.txt`: real filesystem init test output.
- `command-report-json-tests.txt`: deterministic report test output.
- `text-projection-tests.txt`: plain text projection test output.
- `governance-boundary-command-tests.txt`: SDD/Governance boundary test output.
- `full-test-suite.txt`: full solution test output.
- `synthetic-evidence-review.txt`: disclosure review for fixture placeholders.

Later lifecycle-command stories must add evidence for charter, specify, clarify,
checklist, plan, tasks, analyze, generated-view refresh, and analysis reports
before their `[US*]` tasks can be marked complete.
